# ESP32 水质监测系统

基于 ESP32 的多传感器水质监测系统，可实时监测多项水质参数并上传至 ThingSpeak 平台。

## 功能特性

- **水质参数监测**：
  - NH4+（铵离子）浓度（mg/L）
  - pH 值
  - 浊度（NTU）
  - 水温（DS18B20）
  - TDS（总溶解固体）浓度（ppm，0-900）

- **环境参数监测**：
  - 空气温度（DHT11）
  - 空气湿度（DHT11）
  - 光照强度（BH1750）

- **数据传输**：
  - WiFi 连接（支持多组配置，自动选择最优网络）
  - ThingSpeak 云端数据上传

- **WiFi 智能管理**：
  - 多组 WiFi 配置（最多 5 组）
  - 按信号强度自动选择最优 WiFi
  - 自动重连机制（路由恢复后自动重连）
  - Web 配置界面（无需修改代码）
  - 连接质量监控

## 硬件配置

### ESP32 引脚定义

| 功能 | GPIO | 说明 |
|------|------|------|
| RS485 RX | 16 | 接收数据线 |
| RS485 TX | 17 | 发送数据线 |
| RS485 RE/DE | 4 | 收发方向控制（高电平=发送，低电平=接收） |
| DHT11 数据 | 5 | 温湿度传感器数据线 |
| DS18B20 数据 | 18 | 水温传感器数据线（需要4.7kΩ上拉电阻） |
| TDS 模拟输入 | 34 | TDS传感器模拟输出（仅输入，无上拉） |
| I2C SDA | 21 | BH1750光照传感器数据线（默认） |
| I2C SCL | 22 | BH1750光照传感器时钟线（默认） |

### 传感器连接说明

#### 1. RS485 Modbus 传感器（统一总线）

**重要**：三个 RS485 传感器（氨氮、pH、浊度）共用同一个 RS485 总线，采用总线拓扑结构连接。

**总线连接方式**：
```
ESP32                    RS485 总线
GPIO 16 (RX)  ────────── A+ (所有传感器A+并联)
GPIO 17 (TX)  ────────── B- (所有传感器B-并联)
GPIO 4 (RE/DE) ───────── RE/DE (所有传感器RE/DE并联)
GND           ────────── GND (所有传感器GND并联)
5V/3.3V       ────────── VCC (根据传感器要求供电)
```

**传感器配置**：
- **NH4+ 氨氮传感器**
  - Slave ID: 1
  - Modbus 地址: 0x0000
  - 数据格式: 2个寄存器（32位浮点数）
  - 单位: mg/L

- **pH 传感器**
  - Slave ID: 2
  - Modbus 地址: 0x0000
  - 数据格式: 1个寄存器（16位整数，除以100得到pH值）
  - 单位: pH值（0-14）

- **浊度传感器**
  - Slave ID: 3
  - Modbus 地址: 0x0000
  - 数据格式: 1个寄存器（16位整数，除以10得到NTU值）
  - 单位: NTU

**通信参数**：
- 波特率: 9600
- 数据位: 8
- 停止位: 1
- 校验位: 无
- 协议: Modbus RTU

**接线注意事项**：
- 所有传感器的 A+、B-、GND 必须并联连接到总线
- 总线末端建议加120Ω终端电阻（如果传感器没有内置）
- RE/DE 引脚控制收发方向，所有传感器共用同一个控制信号
- 确保所有传感器使用相同的供电电压（通常5V或12V，根据传感器规格）
- 长距离传输时建议使用屏蔽双绞线

#### 2. I2C 传感器

**BH1750 光照传感器**：
```
ESP32          BH1750
GPIO 21 (SDA) ── SDA
GPIO 22 (SCL) ── SCL
3.3V          ── VCC
GND           ── GND
```
- I2C 地址: 0x23（默认）
- 测量范围: 0-65535 lx
- 供电: 3.3V

#### 3. 单总线传感器

**DS18B20 水温传感器**：
```
ESP32          DS18B20
GPIO 18        ── 数据线
3.3V/5V        ── VCC
GND            ── GND
                └─ 数据线（需要4.7kΩ上拉电阻到VCC）
```
- 测量范围: -55°C 至 +125°C
- 精度: ±0.5°C
- 供电: 3.3V 或 5V（建议5V）
- **重要**：数据线必须加4.7kΩ上拉电阻到VCC

#### 4. 数字传感器

**DHT11 温湿度传感器**：
```
ESP32          DHT11
GPIO 5         ── 数据线
3.3V/5V        ── VCC
GND            ── GND
```
- 温度范围: 0-50°C
- 湿度范围: 20-90% RH
- 供电: 3.3V 或 5V（建议5V）
- 数据线内置上拉电阻，无需外加上拉

#### 5. 模拟量传感器

**TDS 水质传感器（模拟量）**：
```
ESP32          TDS传感器
GPIO 34        ── 模拟输出
3.3V/5V        ── VCC
GND            ── GND
```
- 测量范围: 0-900 ppm
- 输出: 0-3.3V 模拟电压
- ADC 分辨率: 12位（0-4095）
- 供电: 3.3V 或 5V
- **注意**：GPIO 34 只能输入，不能输出，适合模拟量读取

### 供电要求

- **ESP32 开发板**：通过 USB 或外部 5V 供电
- **RS485 传感器**：根据传感器规格（通常5V或12V），需要独立供电
- **其他传感器**：3.3V 或 5V（根据传感器规格）
- **建议**：使用外部电源为所有传感器统一供电，避免USB供电不足

### 接线检查清单

- [ ] RS485 总线：A+、B-、GND 正确并联，RE/DE 连接正确
- [ ] RS485 传感器 Slave ID 设置正确（NH4=1, pH=2, 浊度=3）
- [ ] DS18B20 数据线已加4.7kΩ上拉电阻
- [ ] I2C 传感器（BH1750）SDA/SCL 连接正确
- [ ] 所有传感器供电正常（检查VCC和GND）
- [ ] TDS 传感器连接到GPIO 34（仅输入引脚）
- [ ] 所有GND共地连接

## 配置说明

### WiFi 配置（智能自动配置功能）

系统支持 **多组 WiFi 配置**和**智能自动选择**功能，无需修改代码即可配置 WiFi。

#### 核心功能：
- **多组配置**：最多保存 5 组 WiFi 配置
- **智能选择**：自动按信号强度选择最优 WiFi 连接
- **自动重连**：路由故障恢复后自动重连，无需重启
- **连接监控**：实时监控连接质量和信号强度
- **Web 配置**：通过 Web 界面管理所有 WiFi 配置

#### 自动配置流程：
1. **首次启动**：
   - 系统会尝试连接代码中设置的默认 WiFi（如果存在）
   - 如果连接成功，配置会自动保存
   - 如果连接失败，系统会启动配置模式

2. **配置模式**：
   - 系统会创建一个名为 `WaterQualityMonitor` 的 WiFi 热点（无密码）
   - 使用手机或电脑连接到该热点
   - 在浏览器中访问 `http://192.168.4.1`
   - 在配置页面中：
     - 查看已保存的 WiFi 配置列表
     - 添加新的 WiFi 配置（最多 5 组）
     - 删除不需要的配置
     - 立即尝试连接（按信号强度自动选择最优）
   - 配置会自动保存，下次启动时自动连接

3. **智能连接**：
   - 系统会扫描周围 WiFi 网络
   - 匹配已保存的配置
   - 按信号强度排序（从强到弱）
   - 自动连接信号最强的网络

4. **自动重连**：
   - 路由故障时自动检测并开始重连
   - 路由恢复后立即自动重连
   - 无需重启系统
   - 自动选择信号最强的可用网络

#### 默认 WiFi 配置（可选）：
如果需要设置默认 WiFi（作为备选），可以在代码中修改：
```cpp
const char* default_ssid     = "room-8816-941";
const char* default_password = "04851601";
```

#### Web 配置界面功能：
- `/` - WiFi 配置主页面（添加、查看、删除配置）
- `/status` - 查看系统状态（WiFi 状态、IP、信号强度、已保存配置等）
- `/connect` - 立即尝试连接（按信号强度自动选择）
- `/rescan` - 重新扫描 WiFi 网络
- `/clear` - 清除所有配置

### ThingSpeak 配置
在代码中修改以下参数：
```cpp
unsigned long channelNumber = 1880892UL;
const char* writeAPIKey     = "0UWC02XHIMUUKHGK";
```

### ThingSpeak 字段映射
- Field 1: 空气温度 (°C)
- Field 2: 空气湿度 (%)
- Field 3: 光照强度 (lx)
- Field 4: 水温 (°C)
- Field 5: 浊度 (NTU)
- Field 6: pH 值
- Field 7: NH4+ 浓度 (mg/L) - 氨氮
- Field 8: TDS 浓度 (ppm)

## 工作流程

1. **初始化阶段**：
   - 加载保存的 WiFi 配置
   - 按信号强度自动选择最优 WiFi 连接
   - 初始化所有传感器
   - 配置看门狗定时器（30秒）

2. **主循环**：
   - WiFi 连接状态监控和自动重连
   - 连接质量监控（每30秒）
   - 读取所有传感器数据
   - 处理异常情况（返回 0 值）
   - 上传数据至 ThingSpeak
   - 等待 5 分钟（300秒）后重复

## 注意事项

- 所有 Modbus 读取操作都有超时保护（默认 800ms）
- 系统配置了看门狗定时器，防止程序卡死
- 传感器读取失败时会输出错误代码并返回 0 值
- WiFi 配置会自动保存到 ESP32 的 NVS（非易失性存储）
- 系统会自动按信号强度选择最优 WiFi，无需手动选择
- 路由故障恢复后，系统会自动重连，无需重启

## 串口监视器

系统通过串口（115200 baud）输出详细的运行信息，包括：
- WiFi 连接状态
- 传感器初始化状态
- 实时传感器读数
- ThingSpeak 上传状态

## 故障排除

- **WiFi 连接失败**：检查 SSID 和密码是否正确
- **传感器读取失败**：检查接线和传感器供电
- **ThingSpeak 上传失败**：检查 API Key 和网络连接
- **看门狗复位**：检查是否有传感器长时间无响应
